Style

RoboCup Software uses Google Style guides for the most part, which can be found at this website for c++, and at this website for python. This docs page will cover a basic overview of these rules, as well as some of the differences between google and our style guides.

Our style configs for the code formatting programs that we use are located here.

Overview

  1. Wrap lines at 80 characters
  2. Use 4 Spaces for Indentation
  3. Comment your code, try to provide comments for every method.
    • Doxygen has a specific format for comments. More information on commenting via doxygen can be found here.
  4. Place spaces between operators
    • Ex: 4 + 3, (int) i
  5. Write readable code

Style problems are mostly solved by our formatting scripts, although they can't deal with everything.

Autoformatting

We have two make targets dealing with formatting.

make checkstyle will check to see if your changes are up to style.

make pretty will use an autoformater to try and format your code.

Variable Naming Conventions

C++

1 // File names should be CamelCase
2 MyClass.cpp
3 MyStandaloneHeader.hpp
4 MyHeader.h
5 
6 // Class names are CamelCase
7 MyClassName a;
8 
9 // Variables should be camelCase with a lowercase first letter
10 myVariable;
11 
12 // Private variables should be preceeded with an underscore
13 _privateData;
14 
15 // Where possible, avoid C syntax, and use C++ replacements.
16 printf("Hello world\n"); // bad!
17 cout << "Hello World" << endl; // good.

Python

1 # Filenames (and therefore module names) should be lowercase_and_underscore
2 my_class.py
3 import my_class
4 
5 # Class names should be CamelCase
6 class MyPythonClass():
7 
8 # object variables should be underscore_seperated
9 self.my_class_variable = 5
10 
11 # Class or object variables should be CamelCase
12 ClassVariable = 5
